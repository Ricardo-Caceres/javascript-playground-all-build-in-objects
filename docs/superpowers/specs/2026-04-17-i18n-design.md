# i18n Design ‚Äî JavaScript Playground

**Date:** 2026-04-17  
**Status:** Approved

---

## Problem

The platform is English-only. Adding Spanish (and future languages) requires internationalizing both the UI layer and the exercise content (~1,986 exercises).

---

## Approach

`next-intl` with URL-based locale routing (`/en/...`, `/es/...`). Exercise content translated via an AI script, stored in separate JSON files keyed by slug. English is the source of truth; translations are optional overlays with graceful fallback.

### Localized Exercise Hook

```ts
// src/features/exercises/presentation/hooks/useLocalizedExercise.ts
export function useLocalizedExercise(exercise: Exercise, locale: string): Exercise {
  const translations = exerciseTranslations[locale]?.[exercise.slug]
  if (!translations) return exercise
  return {
    ...exercise,
    title: translations.title ?? exercise.title,
    description: translations.description ?? exercise.description,
    hints: translations.hints ?? exercise.hints,
  }
}
```

## Translation Script

`scripts/translate-exercises.mjs`:

- Reads all exercises from `src/features/exercises/infrastructure/data/`
- Batches requests to OpenAI API (gpt-4o-mini for cost efficiency)
- Outputs `messages/exercises/es.json`
- **Idempotent**: skips slugs already present in the output file
- Translates: `title`, `description`, `hints`, `tests[].description`
- Does NOT translate: `solution`, `initialCode`, `assertion` strings, `slug`, `tags`

Requires `OPENAI_API_KEY` env variable.

---

## What Does NOT Change

- Exercise data files (`src/features/exercises/infrastructure/data/**`) ‚Äî untouched
- Exercise `Exercise` TypeScript type ‚Äî no new fields
- Business logic (executor, test runner, gamification, progress) ‚Äî untouched
- URL slugs (`/exercises/array/map`) ‚Äî locale prefix added but slug unchanged

---

## Adding Future Languages

1. Add locale to `src/i18n/routing.ts`
2. Create `messages/<locale>.json` (copy en.json, translate)
3. Run `node scripts/translate-exercises.mjs --locale <locale>`

---

## Out of Scope

- RTL language support
- Translating `assertion` strings (JavaScript code ‚Äî language-agnostic)
- Per-exercise locale override (all exercises use the same locale)
- Server-side locale persistence (cookie + redirect is sufficient)
